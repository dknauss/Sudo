- **Cache MUST be a class property**, NOT a function-local static. The RESEARCH.md "Proposed implementation" sketch's `static $registry_names` is WRONG (reset_cache() cannot clear a function-local static → leaks across the shared-process unit suite). Use `private static ?array $connector_setting_names_cache = null;` and add `self::$connector_setting_names_cache = null;` to the existing `reset_cache()`.

- **Keep the `function_exists('wp_get_connectors')` guard** — legitimate runtime integration check (WP 6.2 min vs 7.0 feature), NOT a shim. Do not remove.
- **Scope = `authentication.method === 'api_key'` only.** Use `isset()` guards on `authentication.method`/`setting_name`.
- **Union, fail-toward-gating:** gate if registry OR regex matches; regex runs on every evaluation.
- **Audit field-name detail is OUT OF SCOPE** — no audit-hook signature change anywhere.
- **Confined to one method + reset_cache().** The `connectors.update_credentials` rule definition (~:481-494) and `request_contains_connector_api_key()` outer loop (~:1026-1034) are UNCHANGED.
