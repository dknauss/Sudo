<objective>
Build the shared server-side foundation for UI session revocation BEFORE any UI
wiring: promote the private CLI `revoke_all_active_sessions()` enumerator into a
shared `Sudo_Session` method (current-site scope, operator-self exclusion), add a
shared `Sudo_Session::is_session_live()` liveness predicate, and factor the reusable
core of the existing AJAX `handle_revoke_session()` (cap check → self-target guard →
target-liveness precondition → rate limit → `Sudo_Session::deactivate()` → audit hook)
into ONE guard sequence for the per-user revoke path. The bulk revoke-all enumerator
shares the leaf helpers (deactivate(), is_session_live()) but stays gate-free so the CLI
can call it un-gated; the UI revoke-all's cap+rate gating is added in plan 02. This plan
also includes the mandatory Pre-Implementation Design Review gate for the whole phase.

Purpose: Interface-first. The UI handlers in plan 02 are "blueprints, not 'build me
a house'" — they call into the shared method, the shared liveness predicate, and the
factored core defined here, with no codebase scavenger hunt. Landing the shared
contracts first prevents two divergent revocation paths (the failure mode REVK-03
guards against) and gives the row-action visibility gate (plan 02) and the core's
target-expired precondition ONE liveness predicate to share.

Output: A `Sudo_Session::revoke_all_active_sessions()` shared method, a
`Sudo_Session::is_session_live()` shared predicate, a CLI command rewired to use the
former, and a factored revocation core in `class-admin.php` (with a target_expired
outcome) — all under TDD with the existing harness green.

REVK-02 scope note (deviation, authoritative per 24-CONTEXT.md): REQUIREMENTS.md
REVK-02 literally says "network-wide", but 24-CONTEXT.md narrows v4.5 revoke-all to
**current-site scope** (the same enumeration the CLI already uses and the per-site
"Sudo Active" view shows). True cross-site/network-wide enumeration is deferred as
REVK-F3. This plan implements current-site-scoped revoke-all deliberately. Recommend
annotating REQUIREMENTS.md REVK-02 to read "current-site" with the network-wide
portion tracked under REVK-F3.

REVK-05 note: the `target_expired` outcome added to the factored core here is the
producer side of REVK-05's third distinct blocked-path message ("That user no longer
has an active sudo session."). Plan 02 Task 3 selects the message FROM this outcome;
without the producer added here, that message would be unreachable and idempotent
deactivate() would silently route the render-to-click race into the success path.

DESIGN-REVIEW DECISION (objection B — resolved: filter on `> time()`): the current
enumerator matches "has the meta key at all", which DIVERGES from is_session_live()
and from the "Sudo Active (N)" count that plan 02's revoke-all interstitial advertises
(it would act on N+stale sessions). The promoted shared method MUST instead enumerate
only LIVE sessions via a `_wp_sudo_expires > time()` meta_query, so revoke-all acts on
exactly the set the count advertises and matches is_session_live(). This is a
deliberate behavior change to the CLI revoke-all (it no longer garbage-collects stale
expired meta — lazy cleanup already handles that; GC is not the CLI's job). Target shape:
```php
public static function revoke_all_active_sessions( int $exclude_user_id = 0 ): int {
    $user_ids = get_users( array(
        'fields'     => 'ids',
        'meta_query' => array( array( // phpcs:ignore WordPress.DB.SlowDBQuery
            'key'     => self::META_KEY,
            'value'   => time(),
            'compare' => '>',
            'type'    => 'NUMERIC',
        ) ),
        'number'     => -1,
    ) );
    if ( ! is_array( $user_ids ) || empty( $user_ids ) ) { return 0; }
    $count = 0;
    foreach ( $user_ids as $user_id ) {
        $user_id = (int) $user_id;
        if ( $exclude_user_id && $user_id === $exclude_user_id ) { continue; } // operator-self
        Sudo_Session::deactivate( $user_id );
        $count++;
    }
    return $count;
}
```

Existing AJAX handler core to FACTOR (includes/class-admin.php:1672) — the reusable
sequence is the body, NOT the AJAX framing (check_ajax_referer / wp_send_json_*).
DESIGN-REVIEW ORDERING (objection A — security): the CAP check goes FIRST so the core
never leaks target liveness to an unauthorized (capless) caller; then the self-target
guard (objection D — the invariant lives in the core, not scattered across callers);
then the target-liveness precondition; then the rate limit; then teardown:
```php
// 1. cap gate FIRST — never reveal target state to an unauthorized caller
if ( ! wp_sudo_can( 'revoke_wp_sudo_sessions' ) ) { return array( 'outcome' => 'no_cap' ); }
// 2. self-target guard — the core refuses to revoke the operator's own session
if ( $target_user_id === $revoker_user_id ) { return array( 'outcome' => 'self_target' ); }
// 3. target-liveness precondition — browser-independent, mirrors row-action visibility
if ( ! Sudo_Session::is_session_live( $target_user_id ) ) { return array( 'outcome' => 'target_expired' ); }
// 4. per-revoker rate limit (private const REVOKE_RATE_PREFIX = '_wp_sudo_revoke_count_'; REVOKE_RATE_LIMIT = 10)
$rate_key = self::REVOKE_RATE_PREFIX . $revoker_user_id;
$revoke_count = (int) get_transient( $rate_key );
if ( $revoke_count >= self::REVOKE_RATE_LIMIT ) { return array( 'outcome' => 'rate_limited' ); }
set_transient( $rate_key, $revoke_count + 1, HOUR_IN_SECONDS );
// 5. teardown
Sudo_Session::deactivate( $target_user_id );
// 6. audit hook — single reason/surface tag param (no separate $surface)
do_action( 'wp_sudo_session_revoked', $target_user_id, $revoker_user_id, $reason, get_current_blog_id() );
return array( 'outcome' => 'success' );
```
NOTE on ordering rationale: cap-first prevents an info-disclosure leak (a capless
caller must not be able to distinguish target_expired from no_cap). target_expired
still consumes NO rate slot and triggers NO deactivate(), so it stays distinct from the
SUCCESS path that idempotent deactivate() would otherwise make indistinguishable — this
NARROWS (does not close) the render-to-click race.
