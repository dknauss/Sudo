**ABAR-01 — Clicking admin bar timer node deactivates the session:**

Source notes to include: class-admin-bar.php handle_deactivate() — checks GET param wp_sudo_deactivate, verifies nonce, calls Sudo_Session::deactivate($user_id), then wp_safe_redirect(remove_query_arg([...])). The href on the .ab-item is the current page URL + ?wp_sudo_deactivate=1&_wpnonce=NONCE (verified).

Test signature: `test('ABAR-01: clicking admin bar timer node deactivates sudo session', async ({ page, context }) => {`

Steps:
- Locate timerNode as `page.locator('#wp-admin-bar-wp-sudo-active')`.
- Use Promise.all: `page.waitForURL(/wp-admin/, { waitUntil: 'load', timeout: 10_000 })` + `timerNode.click()`. The click follows the href (a standard anchor navigation, not AJAX). PHP processes the deactivation and issues a 302 redirect back. waitForURL resolves after the redirect completes.
- After Promise.all resolves, assert `expect(timerNode).not.toBeVisible()` — admin_bar_node() only adds the node when is_active() returns true; after deactivation it is absent.
- Assert cookie: `const cookies = await context.cookies(); const sudoCookie = cookies.find(c => c.name === 'wp_sudo_token'); expect(sudoCookie).toBeUndefined()` — Sudo_Session::deactivate() expires the cookie (sets it with past expiry).

Pitfall to document: This is a full page navigation (not AJAX). The timerNode.click() follows the href. Do NOT use page.waitForResponse() or page.waitForRequest() — there is no XHR. Use Promise.all([waitForURL, click]) exactly as in the stash-replay pattern (Pitfall D from 08-RESEARCH.md). The waitForURL predicate /wp-admin/ is safe here because we are staying on an /wp-admin/ URL (no ambiguity like the challenge page); the redirect destination is guaranteed to be /wp-admin/.

**ABAR-02 — URL does not change after deactivation click:**

Source notes to include: class-admin-bar.php — `wp_safe_redirect(remove_query_arg(['wp_sudo_deactivate', '_wpnonce']))` strips only the deactivation params from the URL and redirects. Starting from /wp-admin/, the final URL is identical to /wp-admin/ (verified).

Test signature: `test('ABAR-02: URL does not change after admin bar deactivation click', async ({ page }) => {`

Steps:
- Capture `const urlBefore = new URL(page.url())` immediately (beforeEach already navigated to /wp-admin/ with session active).
- Use Promise.all: `page.waitForURL(/wp-admin/, { waitUntil: 'load', timeout: 10_000 })` + `page.locator('#wp-admin-bar-wp-sudo-active').click()`.
- After Promise.all: `const urlAfter = new URL(page.url())`.
- Assert `expect(urlAfter.pathname).toBe(urlBefore.pathname)` — same path.
- Assert `expect(urlAfter.searchParams.has('wp_sudo_deactivate')).toBe(false)` — deactivation param stripped.
- Assert `expect(urlAfter.searchParams.has('_wpnonce')).toBe(false)` — nonce param stripped.

Pitfall to document: There IS a navigation. The URL briefly becomes path + ?wp_sudo_deactivate=1&_wpnonce=X before PHP redirects back. Do NOT read page.url() immediately after click() without awaiting the redirect. The Promise.all / waitForURL pattern ensures the assertion runs only after the redirect has completed and the browser is on the final URL.
